# Contributing to AI Relay

Thank you for contributing to AI Relay.

## Development principles

- Keep the project local-first and privacy-first.
- Request only the minimum browser permissions required.
- Do not introduce telemetry by default.
- Keep provider-specific logic isolated from the core.
- Prefer small, reviewable pull requests.

## Workflow

1. Create or choose an issue.
2. Create a branch from `main`.
3. Make one focused change.
4. Run validation and tests.
5. Open a pull request and link the issue.

## Commit style

Use Conventional Commits where practical:

- `feat:` new behavior
- `fix:` bug fix
- `docs:` documentation
- `test:` tests
- `refactor:` internal change
- `chore:` tooling or maintenance

## Security

Do not open public issues for suspected vulnerabilities. Follow `SECURITY.md`.
